# Definition of Ready

This document defines when a user story is ready to be worked on.

## Checklist

A user story is ready to be worked on when ALL of the following are true:

- [ ] **User story is clear** - The "As a... I want... So that..." format is used and makes sense
- [ ] **Priority is assigned** - Marked as Must Have, Should Have, Could Have, or Won't Have (for MVP)
- [ ] **Effort is estimated** - Sized as Small (S), Medium (M), Large (L), or Extra Large (XL)
- [ ] **Dependencies are identified** - What stories this depends on, what stories this blocks
- [ ] **BDD scenarios are written** - Given/When/Then scenarios for testable behavior
- [ ] **Manual testing steps are written** - When BDD scenarios aren't possible
- [ ] **Acceptance criteria are defined** - Clear, testable criteria for completion
- [ ] **No blockers** - No waiting on external decisions or missing information

## Priority Levels

- **Must Have** - Required for MVP, cannot ship without
- **Should Have** - Important but not critical, ship if possible
- **Could Have** - Nice to have, only if time permits
- **Won't Have (for MVP)** - Deferred to later versions

## Effort Estimates

- **Small (S)** - 1-2 days
- **Medium (M)** - 3-5 days
- **Large (L)** - 1-2 weeks
- **Extra Large (XL)** - More than 2 weeks (should be broken down)
