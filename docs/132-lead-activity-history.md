# US-132 — Lead Activity History

**Capability:** Leads
**Status:** Not Done
**Priority:** Should Have (deferrable to post-MVP)

## User Story

As a real estate agent, I want to see a chronological activity log on each lead so that I can understand the lead's full history without relying on memory or scattered notes.

## Dependencies

- US-034 — Create a New Lead
- US-035 — View Lead List

## Notes

OurCRM is a single-user application; activity entries do not include "who performed this action" attribution.

Activity events are auto-generated by the system when the corresponding actions occur. Users cannot manually add, edit, or delete activity entries — the log is a permanent, read-only audit trail.

## Acceptance Criteria

1. The lead detail view (reached from US-035) includes an Activity section showing all logged events for that lead in newest-first order
2. The following events are automatically logged with a timestamp:
   - Lead created
   - Status changed (shows previous → new value, e.g., "Status: Warm → Hot")
   - Pipeline stage changed (shows previous → new stage)
   - Budget changed (shows new value)
   - Source changed (shows new value)
   - Note added (shows note preview: first 60 characters)
   - Email sent to lead (shows subject)
   - Showing scheduled (shows date and property address)
   - Lead converted (notes the resulting transaction)
   - Lead marked as lost
3. Each activity entry shows a plain-language event description and its timestamp
4. Activity entries are read-only and cannot be deleted
5. A filter above the activity list allows filtering entries by event type (e.g. Status Changed, Note Added, Email Sent, Showing Scheduled)
6. A sort toggle switches the list between newest-first and oldest-first ordering

## Manual Tests

**Story:** [US-121 — Lead Activity History](../docs/121-lead-activity-history.md)

### Activity log is visible on the lead detail view
1. Open any lead's detail view and confirm an Activity section is present
2. Confirm at minimum a "Lead created" entry exists with a timestamp

### Status change is recorded with old and new values
1. Change a lead's status (e.g., from New to Warm)
2. Open the Activity section and confirm a "Status: New → Warm" entry appears at the top with the correct timestamp

### Multiple event types are logged automatically
1. On the same lead: add a note, send an email (US-079), and schedule a showing (US-061)
2. Confirm the Activity section shows entries for each action in reverse chronological order
3. Confirm the note entry shows the first 60 characters of the note text
4. Confirm the email entry shows the email subject

### Activity is read-only
1. View the Activity section and confirm there are no delete or edit controls on any entry
